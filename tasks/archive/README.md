# tasks/archive

This directory stores completed task packages that are kept for traceability.

Archive entries preserve the work instructions that were used at the time, while the maintained outcomes live in `docs/` and `artifacts/`. Do not place active task documents here.

## Archive Rules

- Keep one subdirectory per completed work package
- Preserve historical task documents unless a link fix or archival note is needed
- If follow-up work is required, create a new active task package under `tasks/` instead of reopening the archive in place
