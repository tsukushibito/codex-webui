# Design Review

Review design before moving into implementation planning.

## Review Checklist

- Does each material requirement have a design response?
- Are component boundaries and responsibilities clear?
- Are dependencies and integration points identified?
- For consumed interface changes, are the consuming clients or workflows identified?
- Are impact areas visible?
- Are security, operational, and migration concerns considered when relevant?
- Is the design detailed enough to support task planning?
- Are unresolved design risks captured?

## Passing Conditions

Do not move forward until all of these are true:
- Each material requirement has a corresponding design response.
- Component boundaries and responsibilities are clear enough to start implementation.
- Main dependencies, integration points, and impact areas are identified.
- For consumed interface changes, the design states whether the consumer work is in scope now or deferred to an explicit follow-up.
- Security, operational, and migration concerns are addressed when relevant.
- The design is detailed enough to plan implementation tasks without guessing the main structure.
- No unresolved design issue remains that is likely to block implementation immediately.
