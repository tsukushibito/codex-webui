# Style and conventions
- Repository language is currently prose/spec markdown rather than implementation code.
- Existing spec writing style uses Japanese, structured headings, bullet lists, and snake_case identifiers for API fields/enums.
- Internal/public/common API docs use UTC RFC 3339 timestamps with trailing `Z` and opaque string IDs.
- Event naming convention in specs is `domain.action`.
- Git commit messages must follow Conventional Commits per AGENTS.md.
